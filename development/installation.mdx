## Prerequisites

Before you begin, ensure you have the following installed:

- **Bun** (v1.0 or later) - Fast JavaScript runtime and package manager
- **PostgreSQL** (v14 or later) - Database server
- **Git** - Version control

## Installing Bun

If you don't have Bun installed, you can install it using the official installer:

<CodeGroup>

```bash macOS/Linux
curl -fsSL https://bun.sh/install | bash
```

```powershell Windows
powershell -c "irm bun.sh/install.ps1 | iex"
```

</CodeGroup>

Verify the installation:

```bash
bun --version
```

## Setting Up the Project

<Steps>

<Step title="Clone the repository">

Clone the Makisala repository to your local machine:

```bash
git clone <repository-url>
cd makisala
```

</Step>

<Step title="Install dependencies">

Install all workspace dependencies using Bun:

```bash
bun install
```

This will install dependencies for all workspaces:
- `apps/*` - Next.js applications (makisala, kitasuro)
- `packages/*` - Shared packages (db, resend, ui)
- `tooling/*` - Development tools and configurations

</Step>

<Step title="Set up environment variables">

Copy the environment variables template and configure it:

```bash
cp .env.example .env
```

See the [Environment Configuration](/development/environment) page for details on required variables.

</Step>

<Step title="Set up the database">

Ensure PostgreSQL is running, then set up the database:

```bash
cd packages/db
bun run push
```

See the [Database Setup](/development/database) page for more details.

</Step>

<Step title="Start the development server">

Run the development server for the main application:

```bash
cd apps/makisala
bun run dev
```

The application will be available at [http://localhost:3000](http://localhost:3000)

</Step>

</Steps>

## Workspace Structure

Makisala uses a monorepo structure with Bun workspaces:

```
makisala/
├── apps/
│   ├── makisala/          # Main Next.js application
│   └── kitasuro/          # Secondary application
├── packages/
│   ├── db/                # Database schema and migrations
│   ├── resend/            # Email service integration
│   └── ui/                # Shared UI components
└── tooling/
    └── ...                # Linting, TypeScript configs
```

## Available Scripts

### Root Level

```bash
bun install              # Install all dependencies
```

### Application Level (apps/makisala)

<CodeGroup>

```bash Development
bun run dev              # Start development server
```

```bash Build
bun run build            # Build for production
```

```bash Production
bun run start            # Start production server
```

```bash Linting
bun run lint             # Run ESLint
```

</CodeGroup>

### Database (packages/db)

```bash
bun run generate         # Generate migrations from schema
bun run migrate          # Run migrations
bun run push             # Push schema changes directly
bun run studio           # Open Drizzle Studio
```

## Troubleshooting

### Port Already in Use

If port 3000 is already in use, you can specify a different port:

```bash
PORT=3001 bun run dev
```

### Installation Fails

Try clearing the Bun cache and reinstalling:

```bash
rm -rf node_modules bun.lock
bun install
```

### Database Connection Issues

Ensure PostgreSQL is running and your `DATABASE_URL` is correctly configured:

```bash
psql -U postgres -c "SELECT version();"
```
