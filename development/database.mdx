## Overview

Makisala uses [PostgreSQL](https://www.postgresql.org/) as its database and [Drizzle ORM](https://orm.drizzle.team/) for type-safe database queries and migrations.

The database package is located at `packages/db` and is shared across all applications in the monorepo.

## PostgreSQL Installation

### macOS

<CodeGroup>

```bash Homebrew
brew install postgresql@14
brew services start postgresql@14
```


</CodeGroup>

```bash
sudo apt update

```

### Windows

Download and install from [postgresql.org](https://www.postgresql.org/download/windows/)

## Database Setup

<Steps>

<Step title="Create the database">

Connect to PostgreSQL and create the Makisala database:

```bash
psql -U postgres
```

Then run:

```sql
CREATE DATABASE makisala;
\q
```

Or create it directly from the command line:

```bash
createdb -U postgres makisala
```

</Step>

<Step title="Configure DATABASE_URL">

Add your database connection string to `.env`:

```bash
DATABASE_URL="postgresql://postgres:password@localhost:5432/makisala"
```

Replace `password` with your PostgreSQL password.

</Step>

<Step title="Push the schema">

Navigate to the database package and push the schema:

```bash
cd packages/db
bun run push
```

This will create all tables, indexes, and constraints defined in the schema.

</Step>

<Step title="Apply database triggers (optional)">

Some features use PostgreSQL triggers. Apply them manually:

```bash
psql -U postgres -d makisala -f triggers.sql
```

</Step>

</Steps>

## Working with Drizzle ORM

### Database Client

The database client is exported from `packages/db`:

```typescript
import { db } from '@repo/db'
import { accommodations, tours } from '@repo/db/schema'

// Query example
const allTours = await db.select().from(tours)

// Insert example
await db.insert(accommodations).values({
  name: 'Mountain Resort',
  location: 'Swiss Alps',
  // ...
})
```

### Schema Definition

The schema is defined in `packages/db/src/schema.ts` using Drizzle's schema builder:

```typescript
import { pgTable, text, timestamp, jsonb } from 'drizzle-orm/pg-core'

export const accommodations = pgTable('accommodations', {
  id: text('id').primaryKey(),
  name: text('name').notNull(),
  location: text('location').notNull(),
  amenities: jsonb('amenities'),
  createdAt: timestamp('created_at').defaultNow(),
  // ...
})
```

## Migration Workflow

Drizzle provides two workflows for schema changes:

### 1. Push (Development)

For rapid development, push schema changes directly:

```bash
cd packages/db
bun run push
```

<Warning>
  Push directly applies schema changes without creating migration files. Use only in development.
</Warning>

### 2. Generate Migrations (Production)

For production, generate migration files:

<Steps>

<Step title="Update schema">

Edit `packages/db/src/schema.ts` with your changes.

</Step>

<Step title="Generate migration">

```bash
cd packages/db
bun run generate
```

This creates a new migration file in `drizzle/` with a unique name.

</Step>

<Step title="Review migration">

Check the generated SQL file in `drizzle/` to ensure it's correct.

</Step>

<Step title="Run migration">

```bash
bun run migrate
```

This applies the migration to your database.

</Step>

</Steps>

## Seeding Data

Makisala includes a content seeding script at `packages/db/insert-content.ts`:

```bash
cd packages/db
echo '[{"accommodationId":"...","enhancedDescription":"..."}]' | bun insert-content.ts
```

This script accepts JSON data via stdin and updates accommodation records.

### Custom Seeding

Create your own seed script:

```typescript seed.ts
import { db } from './src'
import { tours, accommodations } from './src/schema'

async function seed() {
  await db.insert(tours).values([
    {
      id: '1',
      tourName: 'Mountain Adventure',
      slug: 'mountain-adventure',
      // ...
    },
  ])
  
  console.log('Seeding complete!')
}

seed()
```

Run it:

```bash
bun run seed.ts
```

## Database Triggers

Some functionality uses PostgreSQL triggers defined in `triggers.sql`:

### Automatic Slug Generation

The `generate_slug_from_name` function automatically creates URL-friendly slugs:

```sql
CREATE TRIGGER set_tour_slug
  BEFORE INSERT OR UPDATE OF tour_name
  ON tours
  FOR EACH ROW
EXECUTE FUNCTION update_tour_slug();
```

When you insert or update a tour, the slug is automatically generated from `tour_name`.

### Applying Triggers

```bash
psql -U postgres -d makisala -f packages/db/triggers.sql
```

<Note>
  Triggers are not managed by Drizzle migrations. Apply them manually after schema setup.
</Note>

## Connection Management

The database client uses connection caching in development:

```typescript packages/db/src/index.ts
const globalForDb = globalThis as unknown as {
  conn: postgres.Sql | undefined
}

const conn = globalForDb.conn ?? postgres(env.DATABASE_URL)
if (process.env.NODE_ENV !== 'production') globalForDb.conn = conn
```

This prevents creating new connections on every Hot Module Reload (HMR) update.

## Troubleshooting

### Connection Refused

If you see `connection refused` errors:

1. Ensure PostgreSQL is running:
   ```bash
   # macOS
   brew services list
   
   # Linux
   sudo systemctl status postgresql
   ```

2. Verify the connection string in `.env`

3. Check PostgreSQL is accepting connections:
   ```bash
   psql -U postgres -c "SELECT version();"
   ```

### Authentication Failed

If authentication fails:

1. Reset your PostgreSQL password:
   ```bash
   psql -U postgres
   ALTER USER postgres PASSWORD 'newpassword';
   ```

2. Update `DATABASE_URL` in `.env`

### Migration Conflicts

If migrations conflict:

1. Check which migrations have been applied:
   ```bash
   psql -U postgres -d makisala -c "SELECT * FROM drizzle.__drizzle_migrations;"
   ```

2. Manually resolve conflicts in the migration files

3. Consider resetting the database in development:
   ```bash
   dropdb -U postgres makisala
   createdb -U postgres makisala
   cd packages/db && bun run push
   ```

### Schema Drift

If your database schema drifts from your code:

```bash
# Regenerate from current schema
cd packages/db
bun run generate
bun run push
```
