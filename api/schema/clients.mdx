## Relations

The `clients` table has the following relationships with other tables:

### organization

Many-to-one relationship with the `organizations` table.

- Each client belongs to one organization
- Links to `organizations.id` via `clients.organizationId`
- Cascade delete: when an organization is deleted, all associated clients are deleted

### proposals

One-to-many relationship with the `proposals` table.

- A client can have multiple proposals
- Proposals reference the client via `proposals.clientId`
- Set null on delete: when a client is deleted, associated proposals have their `clientId` set to null

## Usage Example

Clients are typically created when an organization wants to track potential or confirmed customers for tour proposals. Each client record can be associated with multiple proposals, allowing organizations to maintain a history of all proposals sent to a particular client.
