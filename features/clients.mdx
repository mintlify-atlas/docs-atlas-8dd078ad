## Overview

Makisala includes a lightweight CRM to help you manage client information, track communications, and link proposals to the right travelers.

## Data Model

Client schema from `/workspace/source/packages/db/src/schema.ts:171-183`:

```typescript
export const clients = pgTable('clients', {
  id: uuid('id').defaultRandom().primaryKey(),
  organizationId: uuid('organization_id')
    .notNull()
    .references(() => organizations.id, { onDelete: 'cascade' }),
  name: text('name').notNull(),
  email: text('email'),
  phone: text('phone'),
  countryOfResidence: text('country_of_residence'),
  notes: text('notes'),
  createdAt: timestamp('created_at').defaultNow().notNull(),
  updatedAt: timestamp('updated_at').defaultNow().notNull(),
});
```

## Pagination

The list endpoint uses cursor-based pagination:

```typescript clients.ts:30-44
const limit = input?.limit ?? 10;
const offset = (page - 1) * limit;

const data = await ctx.db
  .select()
  .from(clients)
  .limit(limit + 1)  // Fetch one extra to check for next page
  .offset(offset);

const hasNextPage = data.length > limit;
const clientsData = hasNextPage ? data.slice(0, limit) : data;
```

This approach:
- ✅ Efficient for large datasets
- ✅ Consistent results during concurrent updates
- ✅ Simple page-based navigation

## Validation Rules

<Info>
  **Field Constraints**
  - Name: Required, 1-255 characters
  - Email: Valid email format, max 255 characters
  - Phone: Max 50 characters
  - Country: Max 100 characters
  - Notes: Max 5000 characters
  - Query search: Minimum 2 characters
  - List limit: Maximum 100 per page
</Info>
