## Overview

Makisala provides powerful collaboration tools to help your team work together on proposals. Use comments for client feedback, internal notes for team discussions, and assignments to manage workload.

## Comments (Client-Facing)

Comments are visible to clients and used for feedback and questions on proposals.

### Comment Structure

```typescript schema.ts:840-852
{
  id: "comment-uuid",
  proposalId: "proposal-uuid",
  userId: "user-uuid",        // Optional: authenticated user
  userName: "John Smith",     // Display name
  content: "Could we extend day 3 by half a day?",
  
  // Position on proposal (for visual annotations)
  posX: 25.50,   // Percentage from left
  posY: 40.25,   // Percentage from top
  width: 15.00,  // Optional: highlight width
  height: 8.50,  // Optional: highlight height
  
  status: "open", // or "resolved"
  createdAt: "2024-03-12T10:30:00Z"
}
```

<Info>
  Comments support spatial positioning, allowing clients and teams to annotate specific parts of the proposal visually.
</Info>

### Create Comment

```typescript comments.ts:43-112
const result = await trpc.comments.create({
  proposalId: "proposal-uuid",
  authorName: "Jane Doe",
  content: "Can we add a hot air balloon ride on day 4?",

  posY: 45.2,

```

When a comment is created:
- 📧 **Email notification** sent to organization's notification email
- 📍 **Position stored** for visual display
- 🔔 **Real-time update** in team dashboard

### Delete Notes

Users can only delete their own notes:

```typescript notes.ts:142-160

  throw new TRPCError({ code: 'FORBIDDEN' });

```

<Warning>
  Deleting a parent note cascades to delete all replies due to the foreign key constraint.
</Warning>

### Filter by Assignment


## Team Management

### Organization Members

Team members are managed through the Better Auth organization plugin:

```typescript schema.ts:232-242
export const member = pgTable('member', {
  id: text('id').primaryKey(),
  userId: text('user_id').notNull(),
  organizationId: uuid('organization_id').notNull(),
  role: text('role').notNull().default('member'),  // "admin" or "member"
  createdAt: timestamp('created_at').defaultNow()
});
```

### Roles

**Admin**:
- ✅ Manage team members
- ✅ Assign proposals
- ✅ Access all proposals (bypass filter)
- ✅ Configure organization settings
- ✅ Manage billing

**Member**:
- ✅ Create and edit proposals
- ✅ View assigned proposals
- ✅ Add comments and notes
- ❌ Cannot assign proposals
- ❌ Cannot manage team
