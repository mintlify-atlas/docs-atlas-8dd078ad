## Overview

The Content Library is your centralized hub for managing all reusable content across proposals. Store accommodations, upload images, manage activities, and access destination data.

<Note>
  All content is organization-specific except for shared accommodations from the global database.
</Note>

## Organization Images

Upload and manage your own image library.

### Upload Images

```typescript contentLibrary.ts:136-188
const result = await trpc.contentLibrary.uploadImage({
  name: "serengeti-sunset.jpg",
  type: "image/jpeg",  // image/jpeg, image/png, image/webp, image/gif
  base64: "base64-encoded-image-data"
});

// Returns: { id, name, url }
```

<Info>
  **Upload Limits**
  - Maximum file size: 10MB
  - Allowed formats: JPEG, PNG, WebP, GIF
  - Images are automatically compressed and optimized
  - Feature may be restricted by plan tier
</Info>

### List Organization Images

Cursor-based pagination for infinite scroll:

```typescript contentLibrary.ts:82-134
const result = await trpc.contentLibrary.getOrgImages({
  cursor: undefined,  // or previous cursor
  limit: 20           // max 100
});

const { images, nextCursor } = result;
// images: [{ id, name, key, url, createdAt }, ...]
// nextCursor: "base64-encoded-cursor" or null
```

Cursor encoding uses a combination of ID and timestamp:

```typescript contentLibrary.ts:15-30
function encodeCursor(id: string, createdAt: Date): string {
  return Buffer.from(`${id}:${createdAt.toISOString()}`).toString('base64url');
}
```

This approach ensures:
- ✅ Consistent pagination during concurrent uploads
- ✅ Efficient queries (no OFFSET)
- ✅ Stable sort order

### Delete Images

```typescript contentLibrary.ts:190-217
await trpc.contentLibrary.deleteImage({ imageId: "image-uuid" });

// Deletes from both:
// 1. Cloud storage (R2)
// 2. Database record
```

<Warning>
  Deleted images cannot be recovered. Images currently used in proposals will show broken links.
</Warning>

## Storage Architecture

### Cloud Storage (R2)

Images are stored in Cloudflare R2:

```typescript storage.ts
await uploadToStorage({
  file: buffer,
  contentType: "image/webp",
  key: "organizations/org-id/images/filename.webp",
  visibility: "public"
});
```

Public URLs are generated on-demand:

```typescript
const url = getPublicUrl(bucket, key);
// Returns: "https://cdn.example.com/path/to/image.webp"
```

### Database References

The database stores metadata and references:

```typescript schema.ts:153-161
export const organizationImages = pgTable('organization_images', {
  id: uuid('id').defaultRandom().primaryKey(),
  organizationId: uuid('organization_id').notNull(),
  name: text('name').notNull(),      // Display name
  key: text('key').notNull(),        // Storage path
  createdAt: timestamp('created_at').defaultNow().notNull(),
});
```

## Performance Optimizations

### Image Compression

Automatic optimization reduces bandwidth:

```typescript image-utils.ts
const compressed = await compressImage(rawBuffer);
// Converts to WebP when beneficial
// Reduces quality while maintaining visual fidelity
// Can reduce file sizes by 50-80%
```

### Pagination Strategies

**Accommodations**: Offset-based (traditional)
```typescript
offset = (page - 1) * limit
```
Good for: Smaller datasets, page numbers

**Organization Images**: Cursor-based
```typescript
where: lt(createdAt, cursorDate)
```
Good for: Real-time updates, infinite scroll

### Indexes

```typescript schema.ts:468
index('idx_accommodation_images_accommodation_id').on(table.accommodationId)
```

Optimizes queries for:
- Fetching all images for an accommodation
- Counting accommodation images
- Deleting accommodation images
