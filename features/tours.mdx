## Overview

Tour templates are reusable itineraries that serve as the foundation for client proposals. Build your library once, then customize for each client.

<Info>
  Tours can be organization-specific or shared globally across all Makisala users as templates.
</Info>

## Creating Tours

### Basic Information

Every tour includes essential metadata:

```typescript schema.ts:378-397
{
  tourName: "7-Day Tanzania Safari",
  overview: "Experience the best of Tanzania's wildlife...",
  pricing: "5500.00",
  country: "Tanzania",
  number_of_days: 7,
  img_url: "https://...",
  tags: ["wildlife", "safari", "luxury", "family-friendly"]

```

<ParamField path="tourName" type="string" required>
  The name of your tour template
</ParamField>

<ParamField path="overview" type="string" required>

</ParamField>

<ParamField path="pricing" type="decimal" required>

<ParamField path="country" type="string" required>

## Itinerary Days

Build day-by-day itineraries for your tours.

### Day Structure

```typescript schema.ts:414-424
{
  dayNumber: 1,
  dayTitle: "Arrival in Arusha",
  overview: "Welcome to Tanzania! Upon arrival...",
  national_park_id: "uuid-of-park"

```

<Note>
  Day numbers must be sequential starting from 1. The system validates this on update.
</Note>

### Accommodations


Accommodations come from your content library and include:

## Managing Tours

### List Tours

Retrieve all tours for your organization:

```typescript tours.ts:13-28
const tours = await trpc.tours.list();
// Returns: id, name, days, imageUrl, overview, country, pricing, tags
```

### Update Tours

Modify tour details and itineraries:

```typescript tours.ts:96-184
await trpc.tours.update({
  id: "tour-uuid",
  tourName: "Updated Tour Name",
  overview: "New description",
  pricing: "6500.00",
  tags: ["luxury", "private"],
  itineraries: [
    {
      dayNumber: 1,
      title: "Arrival Day",
      overview: "First day activities",
      national_park_id: "park-uuid",
      accommodation_id: "hotel-uuid"
    }
    // ... more days
  ]
});
```

<Warning>
  Updating itineraries replaces all existing days. Make sure to include the complete itinerary in your update.
</Warning>

### Validation Rules

The system enforces strict validation:

1. **No Duplicate Days**: Each day number must be unique
2. **Sequential Numbering**: Days must be numbered 1, 2, 3... without gaps
3. **Positive Integers**: Day numbers must be positive integers

```typescript tours.ts:133-149
// These validations happen automatically
if (uniqueDayNumbers.size !== dayNumbers.length) {
  throw new Error('Duplicate day numbers not allowed');
}

if (!isSequential) {
  throw new Error('Day numbers must be sequential');
}
```

### Delete Tours

```typescript tours.ts:186-200
await trpc.tours.delete({ id: "tour-uuid" });
```

<Warning>
  Deleting a tour will cascade delete all associated itinerary days and accommodations. Existing proposals based on this tour will NOT be affected.
</Warning>

## Shared Templates

### Browse Global Templates

Access curated tour templates shared across all Makisala users:

```typescript tours.ts:225-240
const templates = await trpc.tours.getSharedTemplates();
// Returns up to 100 global templates
```

Shared templates have `organizationId: null` in the database.

### Clone Templates

Create a copy of any template for your organization:

```typescript tours.ts:242-316
const result = await trpc.tours.cloneTemplate({
  templateId: "template-uuid"
});

// Returns: { success: true, tourId: "new-tour-uuid" }
```

Cloning copies:
- ✅ All tour metadata
- ✅ Complete itinerary (all days)
- ✅ Accommodation assignments
- ✅ Activities and features
- ✅ Reference to original via `clonedFromId`

<Tip>
  Cloned tours are fully independent. Changes to the original template won't affect your copy.
</Tip>
